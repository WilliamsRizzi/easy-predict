Quick start

POST https://easy-predict.com/timeseries
Content-Type:        application/json
PAYMENT-SIGNATURE:   <x402-signed-payment-payload>

  "series": [1.0, 2.3, 4.1, 6.8, 9.2]

Endpoints

POST /timeseries x402 · $0.01 USDC

Accepts {"series": [n, …]} or a bare JSON array (3–1000 numbers). Automatically selects the best model — linear, log1p-linear, last-delta, or mean — by holding out the last point as a validation target, then retrains the winner on the full series. Response includes holdout_errors for all candidates so you can see why the model was chosen. Calls without a valid PAYMENT-SIGNATURE header receive HTTP 402 with the full payment requirements in the response body. (X-PAYMENT is accepted as a legacy alias.)

POST /anomaly-detection x402 · $0.01 USDC

Accepts {"series": [n, …], "threshold": 2.0} or a bare JSON array (3–1000 numbers). Returns a list of anomalous points using z-score detection. Optional threshold (default 2.0, range 0–10) controls sensitivity — lower flags more points.

GET /openapi.json free

OpenAPI 3.1 spec. Paid operations declare the x402 security scheme; free operations declare "security": [].

GET /.well-known/x402 free

Machine-readable x402 resource list with full PaymentRequirements for each paid endpoint. Used by x402scan and other crawlers.

How x402 payment works

1
Your agent sends POST /timeseries or /anomaly-detection with no payment. The API returns HTTP 402 with the exact payment terms (network, asset, amount, recipient) in the accepts array.
2
Your agent signs the payment. Use any x402-compatible wallet or the x402 SDK to create a signed EIP-712 transfer for 0.01 USDC on Base.
3
Your agent resends with PAYMENT-SIGNATURE. Include the base64-encoded signed payload in the PAYMENT-SIGNATURE header (X-PAYMENT also accepted as a legacy alias). The facilitator verifies and settles on-chain.
4
You get the result. HTTP 200 with prediction (timeseries) or anomalies (anomaly-detection).
