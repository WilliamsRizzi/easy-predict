Endpoints

POST /timeseries x402 · $0.01 USDC

Accepts {"series": [n, …]} or a bare JSON array (3–1000 numbers). Returns the next predicted value using log1p linear regression. Calls without a valid X-PAYMENT header receive HTTP 402 with the full payment requirements in the response body.

GET /openapi.json free

OpenAPI 3.1 spec. Paid operations declare the x402 security scheme; free operations declare "security": [].

GET /.well-known/x402 free

Machine-readable x402 resource list with full PaymentRequirements for each paid endpoint. Used by x402scan and other crawlers.

How x402 payment works

1
Your agent sends POST /timeseries with no payment. The API returns HTTP 402 with the exact payment terms (network, asset, amount, recipient) in the accepts array.
2
Your agent signs the payment. Use any x402-compatible wallet or the x402 SDK to create a signed EIP-712 transfer for 0.01 USDC on Base.
3
Your agent resends with X-PAYMENT. Include the base64-encoded signed payload in the X-PAYMENT header. The facilitator verifies and settles on-chain.
4
You get the prediction. HTTP 200 with prediction, slope, and intercept.

Discovery
